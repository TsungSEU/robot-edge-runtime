## 2. 云端产品

### 2.1 Aurora-Cloud-Trainer

**定位**: 模型训练平台

**核心功能**:
- PPO 算法训练
- 分布式训练支持
- 超参数自动调优
- 训练进度监控

**输入**:
- 采集数据 (来自 Aurora-Data-Lake)
- 采集元数据 (来自 Aurora-Edge-Runtime)

**输出**:
- 优化的策略模型 (ONNX 格式)
- 训练报告

### 2.2 Aurora-Model-Hub

**定位**: 模型仓库与管理

**核心功能**:
- 模型版本管理
- 模型元数据管理
- 灰度发布
- A/B 测试

**支持的模型格式**:
- ONNX (推理)
- PyTorch (训练)

### 2.3 Aurora-Data-Lake

**定位**: 数据湖与数据管理

**核心功能**:
- 数据存储 (S3 兼容)
- 数据索引与检索
- 数据质量评估
- 数据标注工具

**存储类型**:
- Rosbag2 原始数据
- 元数据索引
- 标注数据

### 2.4 Aurora-WebUI (规划中)

**定位**: 统一管理界面

**核心功能**:
- 集群管理
- 任务调度
- 数据可视化
- 模型监控

## 3. 边缘产品

### 3.1 Aurora-Edge-Runtime (本项目)

**定位**: 端侧智能数据采集 Runtime

**核心功能**:
- ✅ RL 驱动的路径规划
- ✅ 步态级精准触发
- ✅ 实时数据价值评估
- ✅ 云边协同闭环

**版本**: v1.1.2 (当前)

### 3.2 Aurora-Simulation

**定位**: 仿真测试环境

**支持平台**:
- MuJoCo
- Isaac Sim
- Gazebo

**用途**:
- 算法验证
- Sim2Real 测试
- 快速迭代

### 3.3 Aurora-Monitoring

**定位**: 系统监控平台

**监控内容**:
- 系统性能指标
- 采集统计数据
- 上传状态
- 错误告警

### 3.4 Aurora-OTA

**定位**: 模型与配置更新

**功能**:
- 模型热更新
- 配置下发
- 版本回滚

### 3.5 Aurora-Deployment

**定位**: 自动化部署工具

**支持方式**:
- Docker Compose
- Kubernetes
- systemd

| 阶段 | 时间 | 说明 |
|-----|------|-----|
| 数据采集 | 持续 | 7×24 小时运行 |
| 数据上传 | 实时 | 采集后立即上传 |
| 训练数据集准备 | 小时级 | 每 1000 样本/批 |
| 模型训练 | 小时级 | 取决于数据量 |
| 模型验证 | 分钟级 | 自动化测试 |
| 模型发布 | 分钟级 | 灰度发布 |
| 模型部署 | 实时 | MQTT 推送 |

## 5. Edge-Runtime 的角色

### 5.1 作为"数据生产者"

Aurora-Edge-Runtime 是 Aurora 产品矩阵中的**数据生产者**，负责：

1. **高价值数据采集**
   - RL 驱动的智能规划
   - 步态级精准触发
   - 82% 冗余过滤率

2. **实时数据处理**
   - 环形缓冲录制
   - 触发保存机制
   - 元数据生成

3. **云端协同**
   - S3 数据上传
   - MQTT 元数据上报
   - 模型热更新接收

### 5.2 接口定义

#### 上游接口（接收）

| 产品 | 接口 | 数据流 | 频率 |
|-----|------|-------|------|
| Aurora-Cloud-Trainer | MQTT | ONNX 模型 | 按需 |
| Aurora-Model-Hub | REST API | 模型元数据 | 按需 |
| Aurora-OTA-Manager | MQTT | 配置更新 | 按需 |

#### 下游接口（输出）

| 产品 | 接口 | 数据流 | 频率 |
|-----|------|-------|------|
| Aurora-Data-Lake | S3 API | Rosbag2 | 实时 |
| Aurora-Cloud-Trainer | MQTT | 元数据 | 每次采集 |

### 5.3 核心价值总结

| 价值 | 说明 |
|-----|------|
| **智能采集** | RL 驱动的动态路径规划，提高数据价值密度 |
| **精准触发** | 步态级触发机制，减少冗余数据 82% |
| **端侧推理** | ONNX <10ms 延迟，实时决策 |
| **云边协同** | 自动闭环，模型热更新 |
| **多模态支持** | Auto/Humanoid 双模式 |
