## 1. 核心概念

### 1.1 什么是 Aurora-Edge-Runtime？

Aurora-Edge-Runtime 是一个部署在机器人端侧的智能数据采集系统，通过以下核心技术实现高质量数据采集：

| 技术 | 说明 |
|-----|------|
| **RL 驱动规划** | 使用 PPO 算法动态规划采集路径 |
| **步态级触发** | 基于实际步态相位的精准触发 |
| **数据价值评估** | 多维度评估采集数据的价值 |
| **云边协同** | 边缘推理 + 云端训练自动闭环 |

## 2. 系统功能

### 2.1 智能路径规划

基于强化学习的路径规划，根据数据密度和场景语义优化采集路径。

```
观测空间（Humanoid 模式，43维）
├─ 基座速度: 6 (lin_vel 3 + ang_vel 3)
├─ 位置朝向: 4 (norm_pos 2 + heading 2)
├─ 目标信息: 5 (direction 2 + distance 3)
├─ 数据价值: 8 (8方向扇区扫描)
├─ 障碍距离: 4 (4方向射线检测)
├─ 当前价值: 2 (value + rarity)
├─ 采集状态: 2 (collected_ratio + coverage)
├─ 环境: 2 (terrain_type + obstacle_density)
├─ 步态相位: 1 (sin(2π·φ))
├─ 动作历史: 8 (最近8步 forward_vel)
└─ 剩余预算: 1

推理延迟: <10ms
```

### 2.2 步态触发采集

基于实际步态相位的精准触发，确保数据采集在稳定支撑相进行。

```
触发条件 = 1. 足端着地检测
         && 2. 稳定支撑相（双脚支撑且质心稳定）
         && 3. 步长 ≥ 15cm
         && 4. 采集间隔 ≥ 1s
         && 5. 非重复足迹（1000点历史去重）
```

### 2.3 数据价值评估

多维度评估采集数据的价值，指导路径规划优先级。

```
ValueScore = 0.3 × SpatialRarity
           + 0.15 × TemporalFreshness
           + 0.2 × SceneDiversity
           + 0.15 × QualityScore
           + 0.2 × CoverageContribution
```

### 2.4 云端协同

与 Aurora 云端产品协同，实现自动闭环。

```
┌─────────────────────────────────────────────────────────────┐
│                    Aurora 数据闭环                         │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Aurora-Edge-Runtime (端侧)                                 │
│    ├─ 智能采集                                              │
│    ├─ S3 上传                                              │

│           ↕                                                │
│  Aurora-Data-Lake (云端)                                   │
│    ├─ 数据存储                                              │

│           ↕                                                │
│  Aurora-Cloud-Trainer (云端)                               │
│    ├─ PPO 训练                                              │

│    └─ 模型评估                                              │
│           ↕                                                │
│  Aurora-Model-Hub (云端)                                   │

│           ↕                                                │
```

```
┌─────────────────────────────────────────────────────────────┐
│                    数据采集流程                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 规划阶段                                                │
│     Planner → 路径规划 → 代价地图分析 → 价值评估           │
│                                                             │
│  2. 导航阶段                                                │
│     机器人导航 → 步态监控 → 触发检测                       │

│  5. 上传阶段                                                │
│     S3 分片上传 → 元数据上报 → 云端存储                     │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 4. 运行模式

### 4.1 Auto 模式（自动驾驶）

| 特性 | 说明 |
|-----|------|
| 观测维度 | 25 |
| 动作维度 | 4 (离散动作) |
| 触发方式 | 场景触发 |
| 推理延迟 | <5ms |
| 适用场景 | 自动驾驶车辆、轮式机器人 |

### 4.2 Humanoid 模式（人形机器人）

| 特性 | 说明 |
|-----|------|
| 观测维度 | 43 |
| 动作维度 | 3 (连续速度命令: forward, lateral, angular) |
| 触发方式 | 步态相位触发 |
| 推理延迟 | <10ms |
| 适用场景 | 人形机器人、双足机器人 |

### 4.3 模式对比

| 特性 | Auto 模式 | Humanoid 模式 |
|-----|---------|---------------|
| 观测维度 | 25 | 43 |
| 动作维度 | 4 (离散) | 3 (连续) |
| 触发方式 | 场景触发 | 步态触发 |
| 控制频率 | 20Hz | 10Hz |
| 推理延迟 | <5ms | <10ms |
| 内存占用 | <50MB | <100MB |
