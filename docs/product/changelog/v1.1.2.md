## 新增功能

### 核心功能

#### F001: 步态触发机制（重大更新）

- **基于实际步态相位的精准触发**
  - 采集基于实际足端落点，而非规划路径点
  - 稳定支撑相检测（支撑相中间60%）
  - 82% 冗余过滤率验证

- **足迹历史管理**
  - 支持 1000 点足迹历史
  - 自动去重（10cm 半径）
  - 足迹可视化支持

#### F002: 配置热更新

- 支持 `planner_weights.yaml` 热更新
- 支持 `robot_data_collection.json` 热更新
- 无需重启服务，响应时间 <1s

#### F003: Aurora 产品矩阵集成

- 与 Aurora-Cloud-Trainer 模型训练协同
- 与 Aurora-Model-Hub 模型管理协同
- 与 Aurora-Data-Lake 数据湖协同
- 与 Aurora-OTA 模型更新协同

### 系统功能

#### S001: 双模式支持

- Auto 模式（自动驾驶车辆）
  - 24 维观测空间
  - 2 维动作空间
  - 语义约束导航

- Humanoid 模式（人形机器人）
  - 74 维观测空间
  - 18 维动作空间
  - 步态触发采集

#### S002: 路径可视化

- RViz2 实时路径显示
- 计划路径（绿色线条）
- 已采集路径（红色线条）
- 采集点标记（蓝色圆点）

#### S003: 元数据上报

- MQTT 轻量级元数据上报
- ~20B 元数据大小
- 包含奖励、场景类型、质量评分

## 升级指南

### 从 v1.1.1 升级

#### 方式一: 源码编译

```bash
# 1. 备份当前配置
cp config/planner_weights.yaml config/planner_weights.yaml.bak

# 2. 拉取新代码
git fetch origin
git checkout v1.1.2

# 3. 重新编译
cd build
cmake .. && make -j$(nproc)

# 4. 重启服务
sudo systemctl restart dcp
```

#### 方式二: Docker 升级

```bash
# 拉取新镜像
docker pull aurora/edge-runtime:v1.1.2

# 停止旧容器
docker stop aurora_edge_runtime
docker rm aurora_edge_runtime

# 启动新容器
docker-compose up -d
```

### 回滚指南

如果出现问题，可以回滚到 v1.1.1：

```bash
# 源码版本
git checkout v1.1.1
cd build && make -j$(nproc)

# Docker 版本
docker tag aurora/edge-runtime:v1.1.1 aurora/edge-runtime:latest
docker-compose up -d
```

## 已知问题

### 当前版本已知问题

| ID | 描述 | 影响 | 计划修复版本 |
|----|------|------|-------------|
| #501 | ARM64 平台下偶尔出现推理超时 | 低 | v1.1.3 |
| #502 | 某些网络环境下上传失败 | 中 | v1.1.3 |

### 回避方法

**#501: 推理超时**
```yaml
# 降低推理频率
inference:
  num_inference_threads: 2
```

**#502: 上传失败**
```bash
# 增加重试次数
# app_config.json
"retryCount": 5
```
