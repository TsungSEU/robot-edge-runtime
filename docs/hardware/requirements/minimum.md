# 最低硬件配置

本文档描述 Aurora-Edge-Runtime 系统运行的最低硬件要求。低于此配置可能导致系统运行不稳定或功能受限。

## 计算平台

### CPU

| 参数 | 最低要求 | 说明 |
|------|----------|------|
| 架构 | ARM64 或 x86_64 | 必须支持 NEON (ARM) 或 SSE4.2 (x86) |
| 核心数 | 4 核 | 物理核心 |
| 频率 | 1.5 GHz | 稳定频率 |
| 指令集支持 | NEON/SSE4.2 | ONNX Runtime 推理需求 |

### 内存

| 参数 | 最低要求 | 说明 |
|------|----------|------|
| 容量 | 4 GB DDR4/LPDDR4 | 系统可用内存 |
| 带宽 | 10 GB/s | 影响传感器数据处理速度 |

### 存储

| 参数 | 最低要求 | 说明 |
|------|----------|------|
| 容量 | 50 GB | 包含系统、程序、数据和日志 |
| 类型 | eMMC 5.1 或 SSD | 推荐 eMMC 或 SSD，不支持 SD 卡 |
| 顺序写入 | 50 MB/s | 影响数据采集写入速度 |
| 随机读写 | 5000 IOPS | 影响系统响应速度 |

### 网络

| 参数 | 最低要求 | 说明 |
|------|----------|------|
| 上行带宽 | 1 Mbps | 满足元数据上传需求 |
| 下行带宽 | 10 Mbps | 满足模型更新下载 |
| 延迟 | < 100ms | 影响云端交互响应时间 |

## 机器人平台

### 双足机器人 (人形模式)

| 参数 | 最低要求 |
|------|----------|
| 自由度 (DOF) | 12 (每腿 6) |
| 腿长 | 上腿 30cm + 下腿 30cm |
| 髋宽 | ≥ 8cm |
| 关节编码器 | 12-bit 绝对值编码器 |
| IMU | 6轴 (加速度计 + 陀螺仪) |
| 控制频率 | ≥ 50Hz |

### 轮式机器人 (自动驾驶模式)

| 参数 | 最低要求 |
|------|----------|
| 驱动方式 | 差速或阿克曼转向 |
| 轮径 | 10-15cm |
| 编码器分辨率 | ≥ 512 PPR |
| IMU | 6轴 |
| 最大速度 | ≥ 1.0 m/s |

## 传感器规格

### 必需传感器

#### IMU (惯性测量单元)

| 参数 | 最低要求 | 推荐型号 |
|------|----------|----------|
| 加速度计量程 | ±8g | MPU6050, ICM-20948 |
| 陀螺仪量程 | ±500°/s | MPU6050, ICM-20948 |
| 采样率 | ≥ 100Hz | - |
| 噪声密度 | < 400μg/√Hz | - |
| 接口 | SPI 或 I2C | - |

#### 关节编码器

| 参数 | 最低要求 |
|------|----------|
| 分辨率 | 12-bit (4096 步/圈) |
| 精度 | ±1° |
| 响应延迟 | < 1ms |
| 接口 | SPI, I2C, 或 CAN |

### 可选传感器

| 传感器 | 用途 | 最低规格 |

| GPS | 绝对定位 | CEP < 5m |
| 足力传感器 | 接触检测 | ≥ 50N 量程 |

## 电源要求

| 参数 | 最低要求 |
|------|----------|

| 功耗 | < 30W (不含执行器) |
| 峰值电流 | 5A @ 12V 或 2.5A @ 24V |
| 电源纹波 | < 100mV p-p |

## 性能预期

在最低配置下，系统的性能预期如下：

| 指标 | 预期值 |
|------|--------|
| ONNX 推理延迟 | < 20ms |
| 传感器融合频率 | 50Hz |
| 数据采集触发延迟 | < 100ms |
| 步态检测准确率 | > 90% |

## 升级建议

如果系统在最低配置下运行不稳定，建议优先升级：

1. **内存** → 8GB：显著改善多任务处理能力
2. **存储** → NVMe SSD：提升数据写入速度 3-5 倍
3. **CPU** → 8核 @ 2.0GHz：降低推理延迟至 < 10ms
